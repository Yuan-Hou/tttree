# 文风圣经

这是一部文字冒险互动小说的文风总则。它定义「文字应该呈现成什么样子」,是稳定不变的创作底色。

> 说明:本文档由系统中的多个 agent 共享。负责正式写作的 agent 必须严格遵循;负责剧情规划的 agent 可以忽略本文档的文学性要求,只需知道最终的叙事将以此风格呈现即可。

## 视角与人称

- 默认采用**第二人称**(『你』),让读者即主角,代入其感知与抉择。
- 叙述始终贴着主角的感官与意识展开:读者只能知道主角能感知到的东西,不做全知旁白。

## 总体基调:朴素地把故事讲好

- 首要任务是把故事讲流畅、讲清楚,而不是表演文学性。用平实、精确的语言把事情叙述出来——允许「就这样朴素地把它讲出来」,不必每个瞬间都升华、都上修辞。
- 质感与氛围是顺带的收获,不是优先项。先把这一拍的事情朴素流畅地交代好,氛围会从把事情讲对的过程里自然渗出来,而不是靠刻意经营每一句话堆出来。

## 段落颗粒度:连贯成块

- 段落应当连贯成块:一段承载一个小情境,或一段连续的动作与思绪,让它自然地流淌下来,而不是把每个细微动作各切成一个独立的小段。
- 适度分段——在情境、地点或节奏发生转换时才换段;一段比「一个动作一段」明显要整,但也不必大到一段闷到底。读起来应当像正常的小说,是成块的叙述,而不是一行行被切碎的特写。
- 尤其不要把过渡性的动作(找一样东西、走过去、坐下、推开门)拆成多个特写似的小段——这类动作平实地连在一段里顺下来即可。

## 语言与修辞

- 多数描写用直白的语言把画面交代清楚就好。比喻、明喻这类修辞是稀有的点缀,不是每一段的标配——一整个场景里,至多一两处真正用得上的地方,才值得动用它。
- 不要每段都安一个精心设计的比喻或意象,那会显得刻意、像在炫技,反而冲淡了故事本身。优先具体的名词与动词,让画面能被「看见」,而不是被一层层形容词和譬喻裹住。
- 不替读者下结论,不贴情绪标签(不写「你感到害怕」),而是用身体反应、环境细节和动作,让情绪自己浮现。

## 细节:用在该用的地方

- 在重要的时刻——关键的发现、转折、第一次看清某样东西——落地一两个具体可感的细节,把画面给足。
- 过渡处与日常处则平实带过,不必每一段都均匀用力地堆细节。细节是用来标记「这里重要」的;处处都堆,反而谁都不突出。

## 节奏

- 多数内容连贯成段地往前走。单独的短句、单独成行的一句话,是**稀有**的强调手段,只留给真正的关键转折或重击,不是常态——不要动不动就把一句话单起一行来故作悬念。
- 不拖沓:每一段都要推动感知或情节向前,不原地复述已知信息。要紧处可以放慢、给足笔墨;过渡处则收着些、顺势带过。

## 连贯性

- 尊重既定的世界设定、场景状态与角色状态,绝不引入与上下文矛盾的事实。
- 延续此前已经建立的氛围与因果,让每一拍都像是上一拍的自然结果。

## 边界

- 不输出任何系统性元信息(不提『场景』『状态』『选项』『JSON』等概念),只呈现纯粹的故事文字。
